## 🎯 نظرة عامة

**Investly AI** نظام تلخيص نصي متخصص في النصوص الاستثمارية العربية. يأخذ وصفاً طويلاً لمشروع استثماري (دراسة جدوى، تقارير مالية، مقترحات استثمارية) ويولّد ملخصاً منظماً يحتوي على:

- **الميزانية** — التكلفة الإجمالية
- **آلية التمويل** — (ذاتي / قرض / شراكة / مختلط)
- **مدة التنفيذ** — بالسنوات أو الأشهر
- **العائد الاستثماري (ROI)** — النسبة المئوية
- **فرص العمل** — عدد الوظائف المتوقعة
- **مستوى المخاطر** — (منخفضة / متوسطة / مرتفعة)
- **القطاع والمنطقة الجغرافية**

## 🖥 شرح الواجهة

### تبويب 1 — Summarize (تلخيص النص)

```
┌─────────────────────────────────────────┐
│  📝 Full Project Text                   │
│  ┌───────────────────────────────────┐  │
│  │ الصق نص المشروع الاستثماري هنا.. │  │
│  │ (يدعم Ctrl+C / Ctrl+V / Ctrl+A)  │  │
│  └───────────────────────────────────┘  │
│  [ ⚡ Summarize Now ] [ 📂 Load File ]  │
│                                         │
│  💡 Generated Summary                   │
│  ┌───────────────────────────────────┐  │
│  │ الملخص المُولَّد يظهر هنا        │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

**الاستخدام:**
1. الصق نص المشروع في الحقل العلوي، أو اضغط **Load File** لتحميل ملف `.txt`
2. اضغط **⚡ Summarize Now**
3. انتظر حتى يظهر الملخص في الحقل الأخضر السفلي
4. يمكن نسخ الملخص بـ `Ctrl+C` أو بالزر الأيمن

**اختصارات لوحة المفاتيح:**

| الاختصار | الوظيفة |
|----------|---------|
| `Ctrl+C` | نسخ النص المحدد |
| `Ctrl+V` | لصق من الحافظة |
| `Ctrl+X` | قص النص المحدد |
| `Ctrl+A` | تحديد الكل |
| زر أيمن | قائمة سياقية (نسخ/لصق/قص/تحديد الكل) |

### تبويب 2 — Evaluate (تقييم الملخص)

يقيّم جودة الملخص بمقياسَين مستقلَّين:

**ROUGE-L** — يقيس التشابه النصي بين الملخص المُولَّد وملخص مرجعي صحيح.

**Field Score** — يفحص وجود الحقول الاستثمارية الأساسية الثمانية:

| الحقل | الوزن |
|-------|-------|
| الميزانية | 2.0 |
| العائد (ROI) | 1.5 |
| فرص العمل | 1.5 |
| مدة التنفيذ | 1.0 |
| القطاع | 1.0 |
| المنطقة | 1.0 |
| التمويل | 0.8 |
| مستوى المخاطر | 0.8 |

**تفسير الدرجات:**

| النطاق | التقدير |
|--------|---------|
| 80%+ | ممتاز 🟢 |
| 60-79% | جيد 🔵 |
| 40-59% | مقبول 🟡 |
| أقل من 40% | ضعيف 🔴 |

**الاستخدام:**
1. الصق الملخص في حقل "Summary to Evaluate" أو اضغط **Copy from Summarize**
2. اختياري: الصق ملخصاً مرجعياً لحساب ROUGE
3. اضغط **📊 Evaluate Summary**
4. ستظهر النتائج في لوحة تحتوي على مقياس دائري (PulseGauge) وتفاصيل كل حقل


## 🎓 خطوات التدريب

### الخطوة 1 — توليد البيانات

اضغط **"1. Generate data"** في تبويب Train، أو شغّل:

```bash
cd src
python data_generator.py
```

يُنشئ `data/raw_data.csv` يحتوي على **5,000 عينة** استثمارية عربية تركيبية تشمل:
- 30 قطاعاً (طاقة، عقار، زراعة، ذكاء اصطناعي، موانئ، نفط...)
- 22 منطقة جغرافية ليبية
- نصوص بها "نويز" (جمل حشو) بنسبة 70% لتعليم النموذج تجاهل الكلام الزائد
- ~10% من العينات تحتوي حقولاً ناقصة لتعليم النموذج التعامل مع البيانات غير المكتملة

### الخطوة 2 — المعالجة المسبقة

اضغط **"2. Preprocess"** أو شغّل:

```bash
python preprocessor.py
```

يُنفّذ:
- إزالة التشكيل (diacritics)
- طي الأحرف المكررة
- إزالة الرموز غير العربية
- فلترة العينات القصيرة/الطويلة جداً
- حفظ النتيجة في `data/processed_data.csv`

### الخطوة 3 — التدريب

اضغط **"3. Start training"** أو شغّل:

```bash
python model_trainer.py
```

**إعدادات التدريب** (من `config.py`):

| المعامل | القيمة | الشرح |
|---------|--------|-------|
| `TRAIN_EPOCHS` | 3 | عدد الحقب |
| `TRAIN_BATCH_SIZE` | 1 | حجم الدفعة لكل جهاز |
| `GRAD_ACCUMULATION` | 4 | تراكم التدرج (batch فعّال = 4) |
| `LEARNING_RATE` | 5e-5 | معدل التعلم |
| `WARMUP_RATIO` | 0.06 | نسبة الإحماء |
| `MAX_INPUT_TOKENS` | 512 | أقصى طول للإدخال |
| `MAX_TARGET_TOKENS` | 128 | أقصى طول للملخص |

**ميزات التدريب:**
- يكتشف تلقائياً `bf16` على GPU Ampere+ (RTX 4070) لأداء أفضل
- `EarlyStoppingCallback` بصبر 2 حقبة لتجنب overfitting
- حفظ أفضل نموذج تلقائياً بناءً على ROUGE-L
- استئناف من آخر checkpoint عند الانقطاع

**توقع مدة التدريب على RTX 4070 Laptop (8GB):**

| عدد العينات | عدد الحقب | الوقت التقريبي |
|------------|----------|----------------|
| 5,000 | 3 | ~45-60 دقيقة |

### الخطوة 4 — إعادة تحميل النموذج

بعد اكتمال التدريب، اضغط **"4. Reload model"** لتحميل النموذج المُدرَّب دون إغلاق الواجهة.

## 📁 شرح الملفات

### `config.py`
يحتوي على جميع المسارات والثوابت. **المسار المهم:**
```python
HF_MODEL_NAME = r"D:\...\models\AraT5v2-base-1024"  # مسار النموذج المحلي
MODEL_SAVE_PATH = MODELS_DIR / "summarizer_model"     # مسار حفظ النموذج المُدرَّب
```

### `data_generator.py`
يولّد بيانات تدريب تركيبية (synthetic) بقوالب عربية متنوعة مع إضافة جمل حشو للتعليم. قابل للتوسيع بإضافة قطاعات أو مناطق جديدة في `SECTORS` و `REGIONS`.

### `preprocessor.py`
خط أنابيب تنظيف النص العربي. يُستدعى أيضاً في `summarizer.py` وقت الاستدلال لضمان اتساق المعالجة.

### `model_trainer.py`
يضبط AraT5v2 بـ HuggingFace Transformers. يدعم استئناف التدريب من checkpoint ويحفظ سجل التدريب كـ `training_history.csv`.

### `summarizer.py`
يحمّل النموذج مرة واحدة (Singleton) ويبقيه في الذاكرة. استدعاء `unload_model()` يحرر VRAM قبل التدريب.

### `evaluator.py`
مزيج من ROUGE-L (مقياس علمي) و Field Score (مخصص للاستثمار). يعمل بدون ملخص مرجعي (Field Score فقط) أو مع ملخص مرجعي (ROUGE + Field Score).

### `main_gui.py`
الواجهة الرسومية المبنية بـ Tkinter. تُشغّل كل العمليات في خيوط منفصلة (threading) لعدم تجميد الواجهة.

## 🔧 استكشاف الأخطاء

### المشكلة: Badge يعرض "CPU" بدلاً من GPU

**السبب:** PyTorch مُثبَّت بنسخة CPU في البيئة الافتراضية.

**الحل:**
```bash
.venv\Scripts\activate
python -c "import torch; print(torch.__version__)"
# إذا ظهر +cpu:
pip uninstall torch torchvision torchaudio -y
pip install torch --index-url https://download.pytorch.org/whl/cu124
```

---

### المشكلة: `CUDA out of memory` أثناء التدريب

**الحل:** خفّض `TRAIN_BATCH_SIZE` في `config.py`:
```python
TRAIN_BATCH_SIZE = 1   # أقل استهلاك للذاكرة
GRAD_ACCUMULATION = 8  # عوّض للحفاظ على نفس الـ batch الفعّال
```

---

### المشكلة: النموذج لا يُحمَّل (`FileNotFoundError`)

النموذج المُدرَّب غير موجود بعد. يجب إتمام خطوات التدريب الأربع أولاً، أو التأكد من وجود:
```
models/summarizer_model/config.json
models/summarizer_model/model.safetensors
```

---

### المشكلة: خطأ عند تحميل النموذج الأساسي

```
OSError: Can't load tokenizer for 'UBC-NLP/AraT5v2-base-1024'
```

**الحل:** تأكد من وجود المجلد المحلي أو اتصال الإنترنت للتنزيل التلقائي:
```
models/AraT5v2-base-1024/config.json
models/AraT5v2-base-1024/tokenizer.json
```

---

### المشكلة: النص العربي لا يظهر RTL في الواجهة

تأكد من تثبيت خط **Segoe UI** على النظام (موجود افتراضياً في Windows 10/11).

## 📊 مثال على الاستخدام

**النص المُدخَل:**
```
تشهد مدينة طرابلس نمواً اقتصادياً ملحوظاً في قطاع الطاقة المتجددة،
ومن هنا انبثقت فكرة إطلاق مشروع مجموعة الأفق في هذا القطاع الواعد.
تبلغ الميزانية التقديرية 250 مليون دينار ليبي بمدة تنفيذ 5 سنوات.
آلية التمويل عبر شراكة استثمارية مع مستثمرين محليين ودوليين.
يستهدف المشروع عائداً استثمارياً 18% سنوياً مع توفير 1200 فرصة عمل.
تُصنَّف المخاطر بأنها متوسطة وفقاً لدراسة الجدوى المُعدَّة.
```

**الملخص المُولَّد:**
```
مشروع مجموعة الأفق في طرابلس (الطاقة المتجددة).
التكلفة: 250 مليون دينار | التمويل: شراكة استثمارية | المدة: 5 سنوات.
العائد: 18% | الوظائف: 1200 فرصة | المخاطر: متوسطة.
```

**نتيجة التقييم:**
- Field Score: **92.5%** ✅ ممتاز
- ROUGE-L: **0.74** ✅ ممتاز
- الدرجة الكلية: **83.2%**
